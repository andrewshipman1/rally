9C — what to change

1 · readOnly behavior (primary fix)

  • Add readOnly?: boolean prop to Headliner.tsx, default false.
  • When readOnly={true}: drop role="button" and onClick from the outer card surface. Card becomes a pure display container.
  • Suppress hover / active / press states on the outer card: cursor stays default, no transform, no shadow shift.
  • The embedded <a> CTA ("view site →") is the ONLY interactive element — still opens the source URL in a new tab. No other card-level interactivity.
  • SellHeadliner.tsx passes readOnly={true} and drops the noop onOpen={() => {}} — wrapper becomes a pure prop-adapter.
  • Sketch path (via SketchModules) continues to render with default readOnly={false} — tap opens the edit drawer, unchanged.

  • Visual: cream page → white .module-section surface → 2.5px ink border → 16px radius → yellow decorative cap.

3 · Copy audit

  • Every user-facing string in Headliner.tsx resolves via getCopy → lib/copy/surfaces/builder-state.ts (builderState.headliner.*).
