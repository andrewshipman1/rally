9D — what to build

1 · New component

  • Create src/components/trip/CountdownScoreboard.tsx (or equivalent path alongside ChassisCountdown).
  • Mark 'use client' — needs a 1Hz useEffect interval for the seconds tick.
  • Props: target: string (ISO), kicker: string (e.g., "lock in by"), hint?: string (e.g., "until the plan locks"), hintEmoji?: string (wobbling emoji, e.g., "🔒").
  • Do NOT modify or delete ChassisCountdown.tsx. It stays for any remaining callers (lock/go if not migrated yet, etc.). 9D's scope is sell only per Andrew's single-module discipline.

2 · Live tick

  • setInterval at 1000ms. On each tick, recompute and render all four tiles. Don't only tick the seconds tile — if the minute rolls over, hrs/days should roll naturally.
  • Hydration-safe: initial state null, render "--" placeholder until mount (same pattern as ChassisCountdown).
  • Clear the interval on unmount.

3 · Tile treatment (CSS)

  • Each tile: yellow var(--accent) bg, 2.5px ink border, 12px radius, 3px×3px press-shadow. Georgia italic tabular numerals.
  • 4 tiles equally-spaced, 6px gap, centered in the frame.
  • Scoped CSS in globals.css — .scoreboard / .scoreboard-tiles / .tile / .tile-num / .tile-label. Don't conflict with any existing .countdown class.

4 · Motion

  • Seconds tile: subtle scale bump animation (1s loop) as a visual "tick" cue. See @keyframes tick-bump in this mockup.
  • Lock emoji: 3s infinite ease-in-out rotation loop, -4deg → +4deg. See @keyframes lock-wobble.
  • @media (prefers-reduced-motion: reduce) disables both animations.

5 · Phase wiring in page.tsx

  • Sell phase: <CountdownScoreboard target={cutoffIso} kicker="lock in by" hint="until the plan locks" hintEmoji="🔒" />
  • Lock / go phases: same component, different target (tripStartIso) + themed kicker / hint per theme.strings.countdownSignature.
  • Sketch: untouched. Continues to use existing sketch countdown path (if any).

6 · Secondary countdown decision

  • The live sell page currently renders TWO countdowns: primary ("0 days to lock it in", targeting commit_deadline) and secondary ("39 days until we do it again", targeting date_start).
  • The wireframe has ONE scoreboard. Recommendation: drop the secondary countdown on sell. Single scoreboard per wireframe. On lock phase, the single scoreboard pivots to date_start so no urgency is lost.
  • Escalation trigger: if you disagree or see a reason to keep both, STOP and raise options before deleting the secondary call.
Deadline banner stays. The T-3 / T-0 banner ("today's the day · 0 still holding.") renders immediately below the countdowns today. 9D does NOT touch that. Same file, different surface — leave it in place. Its behavior already depends on daysToDeadline, which the new scoreboard data doesn't break.
