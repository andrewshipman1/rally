1 · sticky bar — entry state
Current bar overloads each chip with RSVP_CHIP_ICONS[id] + themed button text (theme.strings.{state}.button). Three pills, three icons, three themed phrases — too much information, especially at 375px. Proposed: keep the global icons, drop the themed text from entry state, replace with the canonical state word. One icon, one word, one decision. Themed flavor moves to the committed pill where it has room to land.

4 · magic-link redirect chain — new-user flow
Andrew observed: brand-new invitee clicks magic link → ends up at /auth/setup instead of the unblur reveal. The architecture was designed to preserve next=/i/<token>?just_authed=1 through the whole chain. Below: each expected step, with the suspected break point marked. Diagnose by reproducing in dev and capturing the URL bar at each redirect.
working step suspect likely break point
1
user on /i/<token> · taps "send me the link"
https://rallyapp.travel/i/abc123
InviteeStickyBar.send() POSTs to /api/auth/magic-link with redirectTo: ${origin}/i/abc123?just_authed=1. Supabase emails the link.
↓
2
user clicks magic link · Supabase appends ?code=…
https://rallyapp.travel/i/abc123?code=PKCE  ← just_authed=1 dropped?
Suspect #1: Supabase may be replacing the entire querystring with ?code=… instead of appending. If so, just_authed=1 is gone before the resolver sees it. Resolver builds nextPath = /i/abc123 (no ?just_authed=1), breaking the freshAuth gate later.
↓
3
resolver detects code · trampolines to /auth/callback
/auth/callback?code=PKCE&next=%2Fi%2Fabc123%3Fjust_authed%3D1
i/[token]/page.tsx:77-82 redirects to /auth/callback with the next param URL-encoded. If step 2 dropped just_authed, next here is just /i/abc123.
↓
4
callback exchanges PKCE · new user → /auth/setup
/auth/setup?next=%2Fi%2Fabc123%3Fjust_authed%3D1&trip=memorial-weekend
auth/callback/route.ts:61-66 propagates next if it passed isSafeNextPath. The path /i/...?just_authed=1 starts with / → passes.
↓
5
user fills profile · save → window.location.href = next
https://rallyapp.travel/i/abc123?just_authed=1
ProfileSetup.tsx:79-86 reads next from the query, sanitizes via isSafeNextPath, navigates. Suspect #2: searchParams.get('next') returns the decoded string with ? intact, and window.location.href = '/i/abc123?just_authed=1' should navigate correctly. Verify in browser.
↓
6
resolver: no code · justAuthed=true · user signed in → freshAuth=true
renders <InviteeShellClient freshAuth={true}> · unblur reveal plays
i/[token]/page.tsx:95 → freshAuth = !!user && justAuthed. If just_authed=1 survived the round-trip, freshAuth=true and the same-tab unblur effect runs. Otherwise the resolver falls into if (user && !freshAuth) redirect('/trip/<slug>') and the user lands on the sell page directly, skipping the moment.
Hypothesis to test first: Step 2. If Supabase strips the existing query when appending ?code=, we never had a chance. Fix candidates: (a) put just_authed=1 in the path, not query (e.g. /i/<token>/auth-fresh sub-route — but that violates the no-new-routes rule); (b) detect the missing flag in step 3 and re-inject it (nextPath = `/i/${token}?just_authed=1` unconditionally, since reaching the resolver with a code at all means we're in the same-tab return); (c) read the original redirectTo from a cookie set in step 1. Option (b) is the smallest fix.
5 · decisions for andrew · sign-off list

locked decisions for 10F (Andrew sign-off, 2026-05-01):

  1. Entry state: segmented control. Icon + canonical state word per chip. New global icon set: RSVP_CHIP_ICONS = { in: '🙌', holding: '🙏', out: '👋' } replaces today's { in: '🙌', holding: '🧗', out: '—' }. One file edit (lib/copy/surfaces/rsvp.ts); affects every surface that reads RSVP icons.

CC strips trailing pictograph from the themed string when it matches the chip icon (handles 🙏 duplicate on holding state).

  4. Haptic: navigator.vibrate(20), guarded. Android Chrome only — iOS Safari silently no-ops. No UI.
  5. Email lexicon wire-up + voice fixes (10F). No HTML shell redesign. Subject + body to getCopy. sendInviteEmail gains themeId + daysOut.
  6. Themed email redesign → 10G (separate session). Per-theme palette + typography + content. Mockups across ≥3 themes first.
  7. Magic-link redirect: diagnose first, smallest patch. Hypothesis: Supabase strips existing query on PKCE append. Fix candidate is ~1 line in i/[token]/page.tsx. Hard escalation if >5 lines or >3 files.
  8. Lexicon scope: 1 new key + 2 icon updates. New: inviteeState.inviteeStickyBar.change = 'change'. Updates: RSVP_CHIP_ICONS.holding ('🧗' → '🙏'), RSVP_CHIP_ICONS.out ('—' → '👋'). All other strings reuse existing keys.
