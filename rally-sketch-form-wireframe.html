Date "When" Field Behavior

• The "when" field contains two native <input type="date"> elements styled as one field
• Display shows formatted dates: "May 29 → Jun 1" (not raw yyyy-mm-dd)
• Tapping the field opens the first (start) native date picker
• After selecting start, auto-focus the end date picker
• End date min attribute = start date value (can't end before you start)
• If no dates selected, show placeholder: "pick your dates"
• DB storage unchanged: date_start and date_end remain separate columns
• Autosave queues both values on change (same as current behavior)

RSVP By Constraints

• RSVP By max attribute = day before date_start
• If no start date set, RSVP By still works as a normal date picker (no constraint)
• If start date moves to before current RSVP By value, clear RSVP By
• Placeholder shows "set a deadline" when empty

Layout Changes in SketchHeader.tsx

Current structure:
field-row: [start] [end] [where]
[rsvp-by]

New structure:
field-row: [when (wide)] [rsvp-by]
[where]

The "when" field replaces start + end. It's wider (flex: 1.5 or similar) to accommodate the range display. RSVP By moves up next to it. Where drops to its own full-width row.

Crew Section — Solid Border + Collapse + Bottom Drawer

• Replace border: 2px dashed with border: 2px solid var(--ink, #1a1a1a)
• Add collapse/expand toggle (chevron icon) in crew header — toggles crew list visibility
• Collapsed state: shows header row only (title + count + chevron). Expanded: full member list
• Default state: expanded
• The "+" button opens a bottom drawer (same component pattern as ThemePickerSheet)
• Drawer contains: name/email input field + send invite button
• On save → drawer closes, crew list updates, section auto-expands if collapsed

Lodging — Solid Border + Collapse + Bottom Drawer

• Lodging section gets the same treatment as crew: solid border container + collapse chevron
• Collapsed state: header only (title + count + chevron). Expanded: cards + add button
• "+ add another spot" opens a bottom drawer (same pattern as crew add)
• Drawer contains: type picker (hotel/airbnb/hostel) + name/URL field + save button
• On save → drawer closes, new lodging card appears in list, section auto-expands
• Replaces current behavior (inline or navigation-based add)

Hotel Badge Z-Index Fix

• The lodging type badge (e.g. "hotel") currently floats over the sticky publish bar at the bottom
• Fix: ensure lodging cards have position: relative; z-index: 1 or lower, and the sticky bar uses a higher z-index
• Also verify at 375px that badge doesn't overlap any other fixed/sticky element

"1 option" Pluralization Fix

• In SketchModules.tsx or builder-state.ts copy key
• When count === 1, display "1 option" not "1 options"
• Could be a conditional in JSX or a pluralized copy key

Mobile-First Emphasis

• All new components (drawers, collapse toggle, field layout) must be designed for 375px first
• Bottom drawers: full-width, touch-friendly targets (min 44px tap), drag-to-dismiss
• Collapse toggle: large enough tap target (min 44px including padding)
• Field row (WHEN + RSVP BY): ensure both fields remain readable at 375px — don't let text truncate
