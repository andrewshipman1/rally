Design Decisions

• "your info" card is new — groups all contact/social fields in one bordered card below the stats. Gives the editable fields a clear home without cluttering the profile head.
• Inline edit pattern: tap any editable field → it transforms into a bordered input with a label. Save on blur or Enter. No save button needed — auto-save with "✓ saved" indicator. Failed saves show red "✗ retry" instead.
• Avatar upload: camera badge on avatar is the tap target. Opens native file picker. Upload happens immediately, avatar updates in-place.
• Phone is read-only with "connected via sms" label — makes it clear this is the auth-linked number, not a casual field.
• @ stripping: social handle inputs strip leading @ on save. Placeholder says "handle without @" to set expectation.
• Back link: "← my trips" replaces needing the browser back button. Links to dashboard (/).

What's NOT Changing

• Stat strip (trips done, ride or dies, countries) — read-only, auto-calculated
• Stamps section — read-only, populated from completed trips
• Ride or dies section — read-only, auto-calculated from shared trips
• Sticky CTA at bottom

Interaction States (for Claude Code)

• Default: all fields in read mode. Pencil icon appears on hover (desktop) or always visible (mobile) for editable fields in the details card. Name and bio show subtle underline on hover.
• Editing: field transforms to bordered input. Focus is set automatically. Label appears at left. Input is full-width within the container.
• Saving: on blur or Enter, call updateProfile server action. Show "✓ saved" in green on success. Show "✗ retry" in red on failure.
• Avatar uploading: after file selection, show loading spinner on avatar while uploading. On success, swap to new image. On failure, revert and show toast or inline error.
