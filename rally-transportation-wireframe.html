design rules baked into this wireframe

tag list (7, final)

tag definition shown in drawer default split
flight intra-trip flight — rome → barcelona, small charter. not your flight to the meetup. individual
train intra-trip train ticket — amtrak, tgv. not your train in. individual
rental car/van car, suv, van, or rv the crew drives on the trip. group split
charter van/bus hired van, shuttle, or bus the crew rides together. group split
charter boat boat, yacht, or fishing charter the crew takes together. group split
ferry scheduled ferry crossing during the trip. individual (override to group split if driving on)
other anything else pre-booked. pick the split that fits. group split

form contract

  • description — required, free text. No character limit (soft UI truncation with ellipsis on the card).
  • type — required, chip picker. Exactly one selected. No multi-select, no blank.
  • estimated cost — required, integer USD. $ prefix is decoration only.
  • split — required, toggle. Default set by tag; freely overridable. Once organizer overrides, don't auto-reset on tag change.
  • link — optional, free text. Enrichment runs on paste/blur via existing /api/enrich. Preview is drawer-only. If enrichment returns nothing, preview block is hidden.

card contract

  • Grid: 28px icon · 1fr body · auto link-chip. Single line.
  • Title = description. Meta = $X · {split} · {tag name}.
  • No hero image. No description preview. No enrichment thumbnail.
  • Link chip (↗) renders only when a URL is present.
  • Tap card → opens drawer in edit mode.

drawer framing (copy contract)

  • Always-visible line above fields: "for what the crew books together on the trip — not how you're getting to the meetup. (the home → meetup leg lives under 'getting here.')"
  • Empty-state hint in collapsed section: "add the stuff the crew books together on the trip — rentals, charters, intra-trip flights or trains."
  • All strings via getCopy. Register new keys in rally-microcopy-lexicon-v0.md §5.29 (transportation).

split semantics (cost summary)

  • individual → contributes cost directly to per-person total.
  • group split → contributes cost / in_crew_count to per-person total.
  • Sketch does not model "which subset of the crew is in on this line." That's a sell/go problem. Current behavior: everyone in-crew is assumed to split.

what this wireframe does NOT mock

  • Loading state of the enrichment preview — implement with the existing .loading shimmer pattern from the headliner drawer.
  • Remove-with-confirm flow — reuse the headliner drawer's confirm-bar pattern (red-tint inline confirmation on tap).
  • The "getting here" slot beyond its helper text. That slot is already shipped in 8I and untouched here.
  • Cost summary surface. 8M wires the math; rendering lives elsewhere and is not this wireframe's concern.

explicit non-goals for 8M (same as 8I + additions)

  • No hero image on the card.
  • No "who's on this" / subset split UI.
  • No dates or assignees on line items.
  • No Google Flights deep-link or flight pricing API.
  • No multi-leg lodging work.
  • No changes to the "getting here" slot.
  • No changes to any other module.
