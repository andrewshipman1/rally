design system — inherited from v2 mockup

tokens (no new ones)

  • --bg: #faf8f5 (cream page background)
  • --ink: #1a1a1a (text + border)
  • --accent: #ffd84d (marquee, highlights — unused in this module)
  • --muted: #888 (labels, hints, dim suffix)
  • --surface: #fff (estimate-row background)
  • --border: #1a1a1a (2.5px row border)

section shell

  • Standard .section: 18×16 padding, 1px ink-12% bottom border.
  • .section-head: title (11px lowercase letter-spaced .12em bold) + count ("3 · estimate" — 11px muted).
  • Single .section-hint below the head replaces the per-row hint that used to sit between activities and provisions.

estimate-row (reuses v2 primitive verbatim)

  • White surface, 2.5px ink border, 12px radius, 14px padding.
  • Label: 11px muted, 6px bottom margin.
  • Value: Georgia italic bold 22px ink. Suffix / person in 14px muted normal-weight sans (via .dim class).
  • Null state: value color switches to muted, number becomes $—.
  • Optional hint below value: 11px muted, 6px top margin.
  • 10px gap between rows. Last row no margin.

number formatting

  • Display via value.toLocaleString() → $50,000, $1,200, $250.
  • Input element stays bare numeric (no formatting while typing). Format on blur / display only.
  • No tilde prefix — v2 uses plain $. Keeps consistency with lodging/transport cost chips.

data wiring (no migration)

  • Activities → existing trips.activities_estimate_per_person_cents.
  • Provisions → existing Groceries.name = 'Provisions' row pattern.
  • Other → NEW Groceries.name = 'Other' row, same save pipeline as provisions.
  • Cost summary: all three contribute as per-person × crew. Line renders in summary as $X per person for each.

lexicon adds

  • builderState.everythingElse.title → "everything else"
  • builderState.everythingElse.sectionHint → "per-person budget for stuff you book ahead of the trip"
  • builderState.everythingElse.activitiesLabel → "activities"
  • builderState.everythingElse.provisionsLabel → "provisions"
  • builderState.everythingElse.provisionsHint → "groceries, snacks, drinks — the stuff you stock up on"
  • builderState.everythingElse.otherLabel → "other"
  • builderState.everythingElse.otherHint → "gifts, decor, entry fees — whatever else"
  • Update builderState.moduleProvisions → "provisions" (was "food & drink"). May be deprecated once new keys land.

out of scope

  • No drawers. No line items. No per-item splits.
  • No new columns on trips. No migration.
  • No cost_unit toggle — all three are per-person.
  • No theme variation — the module is theme-agnostic like the v2 shape.
