Title punctuation accent — implementation

Rendering

JSX-level split. In PostcardHero.tsx, replace:

<h1 className="title">{tripName}</h1>

with:

const { base, accent } = splitTitleAccent(tripName);
<h1 className={titleLengthClass(tripName)}>
  {base}
  {accent && <span className="title-accent">{accent}</span>}
</h1>

Helper:

function splitTitleAccent(name: string) {
  const m = name.match(/([!?.]+)$/);
  if (!m) return { base: name, accent: '' };
  return {
    base: name.slice(0, m.index),
    accent: m[1],
  };
}

function titleLengthClass(name: string) {
  if (name.length <= 16) return 'title t-short';
  if (name.length <= 24) return 'title t-medium';
  return 'title t-long';
}

CSS

.chassis .title .title-accent {
  color: var(--hot);
}

Accent applies only when trailing !?. is present. Long names without trailing punct (e.g., "cape cod for the fourth of july") get no accent — clean fallback, no ugly empty span.

Why --hot not --accent?

  • --accent is yellow; on the cream page --bg it's low-contrast.
  • --hot is the theme's accent red/coral/pink; it's already used for the logo and scrap markers, so it's the established "attention" color.
  • Every theme declares --hot; row 3 above shows it adapting across default (coral red), bachelorette (hot pink), and beach (sky blue) skins.
