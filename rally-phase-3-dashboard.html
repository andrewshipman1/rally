phase 3 · the dashboard · v2 (gamified)

the dashboard is
a game board.

The dashboard isn't a list of trips — it's a board where each tile shows your move. Cards push toward RSVPs the way a game pushes toward your next turn: progress meters, urgent stamps, pulsing borders on the trips that need you, and a scoreboard at the top.

Two pressure tests in one file: light mode and dark mode, both built off the same dual-mode kernel from Phase 2.

What changed from v1

Killed the badges. "live / sketch / locked / done" labels were calendar-app coded. Replaced with countdown stamps in the top right of each card — same visual logic as the trip page hero countdown, just shrunk to card scale. Each stamp signals state through color (yellow = future, lime = locked, red = urgent, grey = sketch, outlined = done). The number does the scanning work the badge label was trying to do.

Added the rally meter. Sell-state cards (where the organizer is collecting yes's) now show a progress bar — "5 of 8 ride or dies." It's the gamification beat: the bar fills as RSVPs come in. Diagonal stripes when the trip needs your move.

Added the scoreboard. Below the H1, a row of chips: "1 your move · 3 cooking · 1 locked · 1 done." It turns the dashboard state into a single scannable line and gives the page a top-of-game-screen feel.

"your move" cards pulse. Cards needing immediate organizer action have a pulsing accent halo and a button-style action ("nudge them →"). The eye finds them first.

Header copy fix. "what's next?" → "where to next? ✈️" per your note.

Dark-mode contrast fixes. Card border now uses --ink instead of --stroke so it stays visible against the dark page bg. Card meta opacity bumped from 0.75 to 0.85. Live-row from 0.65 to 0.78.
