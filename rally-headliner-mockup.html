design notes

  • placement: headliner sits directly under the trip title + meta row. above all cost modules. always the first expense in the cost summary.
  • null state (see tulum): a subtle dashed "+ the headliner (optional)" affordance. non-intrusive. doesn't visually compete with modules. organizers ignore it on destination trips.
  • data shape: description (req) · estimated cost (req) · per-person vs. total flag (req) · optional link. one headliner per trip max.
  • retreat edge case: when the headliner bundles lodging (como, yoga retreats, cruises), the spot module renders as "included in headliner" and skips its own input. worth confirming if this lives in 8j or deferred.
  • activities module: simplified to a single per-person estimate field. no more line items in sketch. matches the provisions shape. full activity line-items + voting move to sell/lock (session 11ish).
  • cost summary: headliner pulls to the top with its own icon + emphasis. reinforces "this is what the trip is about."
