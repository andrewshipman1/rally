▼ cost summary
Theme tokens only — no hardcoded colors. Invite button picks up var(--ink) (deep brown on Coachella); tally pills use color-mix(var(--ink) 8%, transparent) for the subtle surface; empty-state muted text is var(--muted). Works across every theme the app supports without per-theme overrides.

Two design questions, orthogonal to each other

Question 1: container (rows 2 vs 3). Does crew sit in a bordered .module-section like its siblings (Path A), or stay borderless by design (Path B)?

  • Path A (bordered) — crew reads as "another trip page module." Visually consistent with headliner / spot / transport / everything-else. The container signals equal weight.
  • Path B (borderless) — crew reads as "the humans section." The absence of a border makes it feel less mechanical. Aligns with a design philosophy where "content about people" doesn't belong in the same container as "content about stuff."

Question 2: scale (row 4). What happens when a crew has 10+ people? Paths A and B both render every row vertically — a 15-person crew creates ~900px of crew alone, dominating the page. Path C collapses each state into a tally-pill + avatar-pile + chevron header. Tap to expand. At scale the whole module is ~220px tall (collapsed) vs ~760px (all expanded). Same shape at 3 people or 30.

  • Path C can layer on top of A or B. The collapsible behavior is independent of the container choice. You could have "bordered + collapsible" (shown in row 4) or "borderless + collapsible."
  • Mobile-scroll friendly. 375px viewport users don't want to thumb past 12 avatar rows to see cost summary.
  • Signals "there's more" explicitly. The "+4" pile and chevron are unambiguous — you know there are 9 in, you see the first 5 as avatars, tap to see all.

Current render (row 1) has Path B's borderless stance with rough typography and a loud tally treatment, and no collapsibility. Three aspects that can be picked independently.

What's shared across both paths

  • Title + eyebrow treatment. "the crew" at 18px Georgia italic lowercase, matching siblings. Eyebrow "N rallied" in Caveat at 0.5 opacity, matching "rough estimate" / "your way in" style.
  • Tally pills softened. The current solid-ink-with-bg-color treatment (same loud styling we retired from transport count in 9K) becomes subtle ink-on-ink pills. Same data, lower visual volume.
  • Row structure. 36px avatar circle (photo or initial on var(--sticker-bg)), name + host/you markers inline, subtext with RSVP timing / plus-one / decline reason. Same as today.
  • Empty states. Small italic muted text: "everyone's decided" / "nobody's out — knock on wood 👌". Same lexicon keys as today.
  • Invite button REMOVED from this session (Andrew, 2026-04-22). The end-to-end invite flow isn't fully mapped yet (invite delivery is scheduled for Session 11). Pulling the "copy the invite link" button out of the crew module until the flow is sound. When invite delivery lands, the button comes back in a deliberate design pass — likely not as a pill at the bottom of crew.
  • All themed via tokens. No raw hex/rgba. color-mix(var(--ink) N%, transparent) for alpha steps. Works across themes.

What changes from today (either path)

  • Title size. 24px → 18px — matches siblings, stops crew from visually dominating cost summary above it.
  • Subtitle font / weight. Caveat at 0.5 opacity instead of 0.7 — matches sibling eyebrow treatment.
  • Tally pill volume. Solid dark → subtle. No more shouting pills.
  • State-section titles. "in" / "holding" / "out" in 14px Georgia italic. Empty states get a dim class that drops opacity to 0.5, signaling "nobody here" without making the section invisible.
  • Class naming (if Path A). .crew-inline root → wrapped in .module-section.crew-module. Internal .crew-* classes stay.
  • Invite button removed (per above).

Path C specifics (if picked)

  • Default expand behavior. All three states collapsed by default. Exception (optional): "in" auto-expands when ≤3 people total (the whole module fits without scroll anyway — force-collapsing it hides useful info for free). Decide during brief.
  • Interaction. Tap the state-header row (tally pill + pile + chevron) to toggle. State persists per-session via React state — no localStorage.
  • Pile preview size. Mockup shows first 5 avatars + "+N" chip if more. 5 is arbitrary — could be 3 or 7. Flag for brief; depends on what reads well at 375px.
  • Empty state while collapsed. State with zero members — skip the pile, show the empty-state text inline with the tally pill ("0 holding · everyone's decided"). Or hide the state entirely. Decide during brief.
  • Expanded state lazy-load. The mockup's "+ 4 more — still loading…" row is a stand-in. If the server already passes all members (which it does today per page.tsx:487), no lazy-load needed; render all at once. Keep it simple.
  • Accessibility. Collapsible headers should be <button> elements with aria-expanded, not <div> with onClick. Keyboard toggle via space/enter.

Open questions

  • Host marker. Mockup uses 👑 + "host". Current lexicon has crew.hostMarker — check what string it returns and match. Crown emoji might feel too pageant-y; alternatives: star, ★, nothing (just "host" in muted gray).
  • "N rallied" eyebrow copy. Current subtitle text is "N rallied · Coachella 2026!!!". The trip name in the eyebrow feels redundant (trip name is already in the hero above). Proposal: drop it, just "N rallied" — or "N rallied · M total" when invited count differs from rsvp'd count (as row 4 shows). Lexicon change — not unilateral.
  • "Pending" state. Invited but hasn't RSVP'd. Should it render as its own collapsible section ("pending: 2") below "out"? Current code handles it but the mockup shows only in/holding/out — check if pending renders today and match. Especially important with Path C's scale story — pending invitees are a common state.
  • The members={members as any} cast at page.tsx:487. Not a visual issue but signals a type mismatch. Out of scope for this visual session; logged for follow-up as its own bug-bash item.

  3. Open questions above — decide or defer per item.
