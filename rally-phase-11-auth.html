phase 11 · auth / magic link

the door

Every Rally session starts here. Magic link only in v0 — no password, no SSO, no social. The landing is one field and one button. The sent state is a confirmation with a friendly nudge. Errors are named, specific, and always offer a next move.

Flow: invite link → email prompt → magic link sent → user clicks link → authenticated → redirect back to the trip page (§5.17 invitee state becomes §5.4–5.8 full trip page). Existing sessions skip the whole thing.

Constraints regardless of provider:

  • Magic link expires in 15 minutes.
  • Single-use. Clicking a used link → "link invalid" state.
  • Email delivery via Resend (already in lexicon §5.14).
  • Session duration: 30 days rolling (reset on each visit).
  • Resend cooldown: 30 seconds between sends per email.
  • Rate limit: 5 sends per email per hour before lockout.

behavior notes

  • Existing session: skip landing entirely. Tapping an invite link with a live session goes straight to the trip page.
  • Invitee flow: invite link carries a trip_id + invite_token. After magic-link auth, redirect to /trips/{trip_id} and mark the invite as opened. If the user declined pre-login via the §5.17 "can't make it" CTA, record the decline with the "before seeing the plan" asterisk.
  • Loading state: send button swaps to "sending…" and disables during the POST. On success, route to state 2. On failure, toast the error inline above the button.
  • Resend cooldown: the "send another" link becomes "hang on — {n}s" and disables for 30 seconds after each send.
  • Account creation is implicit: first successful magic-link confirm creates the user record (email only). Display name is collected later on the profile page (§5.15), not at the door.
  • No password recovery flow — there are no passwords. If the user can't get an email, they're locked out until they can.
