phase 6 · theme picker · g8

try on a vibe.

Organizer opens the theme picker from the trip page in Sketch phase ("change vibe" link on the live-row). A bottom sheet slides up with all 17 themes as chassis swatches, categorized by occasion / setting / default. Tap a tile to preview in place — the trip page behind the sheet re-skins live so the organizer sees what they'd ship. Tap "lock it in" to commit. Selecting a theme is not just a palette change — it swaps all 20 content strings (stickers, marquee, hints, RSVP flavor, nudges, CTA). See rally-theme-content-system.md for the full string set.

Interaction spec

  • Entry point: "change vibe" text link on the live-row in Sketch phase. Becomes a small icon in Sell/Lock/Go phases (still accessible, less prominent).
  • Sheet: bottom sheet, ~82% of viewport height. Dimmed blurred trip page behind.
  • Category tabs: "all · occasion · setting · default". Default "all" shows everything in one grid, ordered as listed in the content system doc.
  • Preview in place: on tap, the dimmed trip page re-renders with the new palette + new strings live, while the sheet stays up. Feedback is instant — no delay, no apply button between taps. This is the flex moment.
  • Commit: "lock it in" button in the confirm bar at bottom of sheet. Closes sheet and applies. Until committed, the theme is a preview and backs out if the sheet is dismissed.
  • Dismiss: drag the handle down, tap the blurred area, or tap a system back gesture. Reverts to previously committed theme.
  • Initial state: on a new trip, picker opens automatically with "just because" pre-selected. Organizer picks before moving on.
  • Editing later: can be re-opened any time. Changing theme after lock/go is fine — it's just a vibe swap, no data loss.

Tile anatomy

Each tile is a compact chassis preview. Three parts:

  • Swatch (top, 78px tall): diagonal gradient of the theme's accent → bg. The wordmark "rally" in Shrikhand, rotated, using the ink color. A mini sticker in the sticker-bg color showing the theme's name or emoji. This is the whole palette story in one 180×78 rectangle.
  • Meta (bottom): theme name in 11px bold, theme type (occasion / setting / default) in 8px uppercase. Ink color from the theme.
  • Selected state: 3px accent-colored shadow around the tile + a small check badge in the top-left corner.

claude code handoff notes

Data model: trip has a theme_id field. All 17 themes are static definitions in a themes.ts registry (palette tokens + string overrides), loaded from rally-theme-content-system.md.

Theme registry shape: { id, name, type, palette: { bg, ink, accent, accent2, stickerBg, surface }, strings: { stickerNew, stickerInvite, ..., ctaSend } }. One flat object per theme.

Apply mechanism: theme tokens are applied as CSS custom properties on a chassis wrapper. No prop drilling — just set --bg, --ink, etc. on the root, and the chassis kernel does the rest.

Preview-in-place: while the picker sheet is open, the dimmed trip page is rendered with the previewed theme (not the committed one). On "lock it in", committed theme becomes previewed theme and the sheet closes. On dismiss without commit, rollback to the previously committed theme.

Strings come from theme, not lexicon overrides: the lexicon has a global default for each string key; each theme overrides. At runtime, look up a string as theme.strings[key] ?? lexicon[key]. The default theme "Just Because" happens to match the lexicon defaults for most keys.

Variable interpolation: theme strings can contain variables like {festival}, {age}, {partner}, etc. See content system doc §"Festival variable strings". Organizer supplies these in the trip setup form. If a variable is missing at render time, render with a graceful fallback ("the birthday" instead of "Amelia's 30th", etc.).

No user-created themes for v0. 17 curated themes only. Custom theme creation is v0.1+.

Category tabs: "all" is the default and the most common case. The occasion/setting/default tabs are navigational only — they filter the grid, don't reorder.
