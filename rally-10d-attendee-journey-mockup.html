Phase 5's design included a "can't make it" opt-out button that recorded out without signup. That path is rejected for v0. Teaser sticky bar shows a single primary CTA ("see the plan →") that routes to the magic-link auth flow.

Rationale (Andrew, locked 2026-04-27): signup is the data capture moment — that's the point of the teaser, not just gating yes/holding. Curiosity drives signup conversion. One consistent gate is simpler UX than splitting yes-from-no friction.

Implementation impact: 10D's InviteeShell diverges from phase 5 on the sticky bar specifically. Rest of phase 5 (blur veil, called-up sticker, countdown, going-row) stays authoritative.

4. tap "see the plan" → auth round-trip reuses phase 11
Phase 11 authoritatively designs four states: landing (state 1) → check email (state 2) → optionally expired (3a) or invalid (3b). Open rally-phase-11-auth.html for the polished frames. Already wired in code: /auth, /auth/expired, /auth/invalid all exist; signInWithOtp via Supabase is already firing.
auth flow at a glance
  • State 1 — Landing: "what's your email?" + email input + "send link" CTA. Renders at /auth.
  • State 2 — Check email: "we sent a link to jen@example.com." + "didn't get it? send another" with 30s cooldown.
  • State 3a — Expired: magic link clicked > 15 min after issue. "this link expired — try again" + restart flow.
  • State 3b — Invalid: tampered or fully invalid token. Same shape, different copy.
  • On successful magic-link click: invitee returns to Rally signed in, hits the auth/callback route, which redirects them — for invitees, redirect target should be /i/<token> so the teaser surfaces with auth context (or directly to /trip/<slug> bypassing the teaser, since they're now authenticated). See open question below.

animation spec (proposed)

Duration: ~600-800ms total. Long enough to feel intentional, short enough not to stall.

Sequence: blur radius animates from blur(8px) → blur(0) over 400ms ease-out. Lock pill fades + scales down (200ms). Sticky bar transitions from teaser shape (can't make it / see the plan) to the full RSVP three-pill bar (in / holding / out) with 200ms cross-fade.

Implementation: CSS transitions on a class swap. No JS animation library needed. Triggered when client-side state flips from teaser to authenticated after the magic-link callback completes.
