The core design question (rows 2 vs 3 vs 4)

  • Path A (bordered, cream). Total is "another sell module." Maximum consistency — same container, same header pattern, same row rhythm as transport / everything-else / crew. Visually lightest. Makes the rollup read as "summary" not "anchor."
  • Path B (refined dark). Preserve the "total is the financial anchor" intent. Dark surface draws the eye. Modernized hierarchy — header + eyebrow like siblings, rows cleaner, badges as footer. No bordered container.
  • Path C (hybrid). Bordered container like siblings, dark hero block integrated at the top. Reads as "rollup module" from the outside, "financial anchor" from the inside. Most deliberate but also most design.
My lean: Path C. It acknowledges both instincts — CostBreakdown IS a sell module (bordered, in the rhythm) AND IS the financial anchor (dark hero draws attention to the number). Path A loses too much weight; Path B stays outside the sibling pattern. Path C threads the needle.

What changes from today (all paths)

  • Hero scale. 52px → 42-46px. Still the biggest number on the page but not oversized. Georgia italic treatment instead of sans bold — matches sibling titles.
  • Per-row progress bars removed. The ~70px track per row added visual noise for marginal benefit. Numbers alone convey proportion when you scan the column. Drops ~8px of vertical per row and dramatically simplifies the render.
  • Row shape. Grid: auto · 1fr · auto for icon · label · value. Matches transport module's compact card shape (9K). Consistent visual grammar across modules.
  • Line labels mirror what the organizer entered above (Andrew, 2026-04-22). Cost rows read as a readable recap of what was set up in each module — "the headliner · Coachella Valley Music & Arts Festival", "lodging · Cap Juluca (so far)", "transport · Saint Barths Plane" if single-item or just "transport" if multi-item. NOT trimmed. The redundancy with the module above is fine — the rollup IS the summary, and the summary should name what it's summarizing.
  • Badges restructured. Pill stack → subtle footer row with ink-colored values. "🏠 Shared: $3,600" + "✈️ Book yours: $4,000" stays conceptually — left/right in a small footer divided by the existing top-border rule.
  • Title "your total" / "per person". Per-viewer uses "your total"; logged-out / non-member uses "per person" (group fallback). Eyebrow copy: "firming up" when pending arrival or unlocked lodging; empty when everything's filled in.

Theme audit (Andrew 2026-04-22 — Path C locked after audit)

Audited all 17 Rally themes for dark-hero viability. All 17 themes ship clean with Path C — but the hero block uses var(--surface) + var(--on-surface), NOT var(--ink) + var(--bg).

Why the token swap matters: 3 themes (boys-trip, city-weekend, festival-run) have a DARK page canvas (bg=near-black/deep-purple) and a LIGHT ink (cream/bone). If Path C used var(--ink) as the hero background, those 3 themes would render a LIGHT hero on a DARK canvas — inverted from the intended "dark hero = financial anchor" visual. The design system already defines --surface as the dark-card color and --on-surface as its text — every theme has this pair as dark/light regardless of canvas. Consistent dark hero in every theme.

  • 16 of 17 themes have strong accent-on-surface contrast (terracotta on brown, pink on charcoal, orange on navy, etc.) — safe for the .emphasize color on "your way in" / "the headliner" rows and the hero-block accent elements.
  • 1 edge case — festival-run: hot pink accent #ff3a8c on deep purple surface #2a1540. Still readable at 42–46px hero scale, but reduced vs other themes. Flag for visual QA check during AC run; if it looks muddy, swap festival-run's accent to an accent2 variant or redefine the surface to a higher-contrast purple. Don't block the session on this.
