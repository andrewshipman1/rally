What this mockup is proving

  • Single parameterized template. Inspect the CSS — one .email ruleset, then three [data-theme="..."] overrides that swap exactly the chassis vars. No forked HTML.
  • Marquee strip up top. Mirrors the in-app trip-page module order (marquee → hero → countdown). Same chassis treatment: --surface block, --sticker-bg text, --accent star separators, DM Sans bold 11px @ 0.18em tracking. Strings pulled directly from each theme's strings.marquee array. Browser mockup animates (18s scroll) so the vibe reads. The shipped email renders STATIC — Gmail/Outlook strip CSS animations; what the recipient sees is the first frame, which is still the same brand-tight strip. Apple Mail will animate it for free.
  • Hero = accent-block + themed sticker + trip name. No cover image; renders identically across Gmail / Outlook / Apple Mail / dark mode (accept-as-limitation: light theme palette will look slightly off in dark mode, documented).
  • Sticker uses sticker.invite from each theme verbatim — birthday-trip "you're invited 🎂", beach-trip "sand szn 🏖️", ski-chalet "send it ❄️". Tilted, sticker-bg behind, stroke border — same shape as the in-app sticker pattern.
  • Subject + body are real — pulled from src/lib/copy/surfaces/emails.ts with theme-id-agnostic copy (10F's voice work; 10G doesn't change it).
  • CTA inherits --accent; footer inherits --surface/--on-surface with a sticker-bg accent dot.
  • What's deliberately not here: 17 template forks; cover image render path; dark-mode media queries; marquee re-render in HTML email.

Open question for the brief

  • Web-safe font: do we keep Georgia for the trip name (current email.ts) or try Fraunces with Georgia fallback? Fraunces likely strips in Outlook, falls to Georgia anyway. Recommendation: keep Georgia, write off Fraunces for email-shaped surfaces.
  • Does the sticker rotate -2deg survive Outlook? Worth testing — fallback is no-rotation (still readable, slightly less brand).
