what else is on the list?

Extras live between itinerary and cost on the trip page. Four types in v0: packing list, playlist, house rules, shared album. None are required. Organizer (or any crew, TBD) taps "+ add something extra" to open the chooser sheet. Left: empty state + chooser open. Right: all four cards populated.

principles

  • All four are cheap. None of them host data Rally doesn't already have a way to handle. Shared album is a link field, playlist is a smart-link card, house rules is text, packing list is a checklist.
  • Playlist and album are smart-linked. Paste a Spotify URL → we pull album art + title. Paste an Apple Photos link → we render the link-out card. If the fetch fails, graceful fallback (domain pill + Caveat label).
  • Tappable targets: playlist art opens the Spotify link. Album icon opens the photo share URL. Packing list items toggle done state. House rules are read-only after the organizer edits.
  • Only show the section if there's at least one extra. Empty state only appears if user has tapped "+ add" from a zero state, as a prompt. Otherwise the extras section is invisible — don't clutter trip pages that don't need it.
  • Shared album is a link, never hosted media. One URL input, one tap-to-open card. Rally does not store images.

claude code handoff notes

Data model: trip has an extras array. Each extra has type (packing/playlist/rules/album), data (type-specific), and added_by.

Packing list: array of { label, done, added_by }. Any crew can add items and toggle their own. Organizer can remove any. No assignments in v0 (not "Amelia brings the charcuterie" — that's v0.1).

Playlist: single URL field. Smart-link pulls title, art, track count, platform (spotify / apple music / youtube). Fallback: paste shows a plain card with the URL + open-link CTA.

House rules: single text blob, markdown-lite (just bullets). Organizer edits, everyone reads.

Shared album: single URL field. Smart-link pulls title if available (Apple Photos shared albums often expose a title via OG tags). Opens externally on tap. Rally does NOT host images.

Add flow: tap chooser tile → open type-specific add form (packing = first-item field, playlist/album = URL field, rules = textarea). One form per type, all stay on the trip page as bottom sheets.

Who can add: organizer always. Crew who've RSVP'd "in" or "holding" can add packing list items. Playlist / rules / album = organizer only in v0 (prevents chaos, room to loosen later).

Removal: organizer can remove any extra. No confirmation for packing items. Confirmation for whole-card removal ("delete the whole playlist card?").

Empty state visibility: the dashed "nothing here yet" card and "+ add" button only render if the organizer has tapped into the extras section at least once. Otherwise the whole extras section is hidden on the trip page for trips that don't use extras.
