9E — what to build

1 · Scoreboard card wrapper (the "object around it")

  • Wrap the existing .scoreboard contents in a new .countdown-card container.
  • Treatment: white var(--surface) background, 2.5px ink border, 14px radius, 3px-3px press shadow. Padding ~16px top/sides, 18px bottom.
  • Keep the existing scoreboard internals (kicker, date, tiles, hint) exactly as they are — 9E is the wrapper only. Tile sizing / tick-rate fixes live in 9D-fix; don't re-fix them here.
  • Scoreboard margin-bottom baked into the card, not the interior.
  • Scope the CSS under .chassis .countdown-card so it doesn't collide with the existing .countdown class that ChassisCountdown uses.

2 · Cover image reposition

  • Today PostcardHero.tsx renders <div className="postcard-cover"> between the marquee and the header block (above the sticker / title).
  • Move it to sit between the hero text stack and the scoreboard card. In practice that's below the tagline/title and above the countdown card.
  • NOTE: 9F will add a trip meta row ("dates · destination") between tagline and postcard. For 9E, place the postcard directly below the tagline — leave the meta-row slot unreserved. 9F will push the postcard down by one row when it ships. No placeholder needed.

3 · Cover image — present variant (Variant A)

  • When trip.cover_image_url is set, render the image at 16:9 aspect ratio inside a .postcard frame.
  • Frame: 2.5px ink border, 12px radius, overflow: hidden.
  • Subtle bottom-gradient tint overlay to keep the stamp legible on bright imagery. See the .postcard--image rule in this mockup.
  • Use the existing next/image <Image> setup as the image source — don't refactor imaging logic.

4 · Theme-gradient fallback (Variant B)

  • When trip.cover_image_url is null, render the same frame with a theme-color gradient instead of an image.
  • Gradient colors come from existing theme tokens. If there's no first-class "theme-gradient" token pair today, escalate before adding. Likely candidates: reuse --accent + --hot or a pair defined in each theme's string file. CC decides after reading the theme token structure; flag in release notes if a new token was added (preferred: none).
  • Same 2.5px ink border + 12px radius. Same stamp pill placement.

5 · Destination-stamp pill

  • Small white pill, 1.5px ink border, 7px radius, slight rotation (~3deg), 1.5px-1.5px mini press shadow.
  • Position: top-right corner of the postcard, 10px from each edge.
  • Content: trip.destination (e.g., "tortola · bvi"). Lowercase, Georgia italic.
  • Hide the stamp if trip.destination is unset.
  • If the stamp copy doesn't have a lexicon home, the raw field is fine — destination is data, not voice copy. No new lexicon entry required.
Escalation trigger: if theme tokens don't cleanly support a two-color gradient, STOP and flag. Options: (a) pick two existing per-theme color vars, (b) add a new --theme-gradient-a / --theme-gradient-b token pair per theme (17 themes to update — material work), (c) fall back to a shared neutral gradient across all themes for v0. Don't invent token names without alignment.
